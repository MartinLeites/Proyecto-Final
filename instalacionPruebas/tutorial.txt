Para hacer funcionar las pruebas de php unit no sera suficiente tener todo descargado, hay que asegurarnos de que tambien se crea la base de datos que se usara para las pruebas y phpunit instalado en composer.
Luego de haber hecho el manual que explica en github tal cual de como hacer funcionar el proyecto, haremos lo siguiente:
1) Nos fijaremos si la BD de pruebas esta creada entrando a phpmyadmin (localhost/8081). SI no esta seguimos con el proximo paso
2) Creamos DB dbSS_test y le damos privilegios al usuario userSS sobre esta BD en el ccontenedor BD
docker exec -it db bash
mysql -u root -proot -e "CREATE DATABASE IF NOT EXISTS dbSS_test;"
mysql -u root -proot -e "GRANT ALL PRIVILEGES ON dbSS_test.* TO 'userSS'@'%'; FLUSH PRIVILEGES;"
3) Ahora debemos instalar phpunit, para eso entramos al contenedor web con
docker exec -it web bash
y ejecutamos:
composer require --dev phpunit/phpunit
4) Sin irnos del contenedor web, hacemos las migraciones faltantes con
php artisan migrate --env=testing

Y listo! podemos hacer pruebas usando: 
./vendor/bin/phpunit
o para ejecutar una especificamente:
./vendor/bin/phpunit tests/ruta/a/prueba.php
